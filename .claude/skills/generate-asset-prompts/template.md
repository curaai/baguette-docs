# 프롬프트 파일 템플릿

## 00-README.txt 스켈레톤

```
{에셋명} 프롬프트 모음 — Nano Banana(Gemini 2.5 Flash Image)용
(ComfyUI 파일도 있으면 "/ ComfyUI용" 추가)

사용법
1. 각 파일 상단 "참조 이미지"에 적힌 파일을 함께 업로드한다.
   경로: Docs/원화_테스트/ (기존 결과물), Docs/assets/아트 기조/ (컨셉아트)
2. "프롬프트:" 아래 영문 문단을 그대로 복사해 붙여넣는다.
3. 결과가 마음에 안 들면 같은 채팅에서 이어서 수정 지시.
4. 마음에 든 결과는 Docs/원화_테스트/ 에 저장하고 파일명에 컨셉 번호를 남긴다.

정본 근거: [[planning/direction/art-direction]] (관련 태스크가 있으면 여기 추가)

파일 목록
- 01-{컨셉명}.txt — 한 줄 설명
- ...
```

## NN-{컨셉명}.txt 스켈레톤 (Nano Banana)

```
[컨셉] {컨셉명}
근거: {어느 문서/이미지에서 이 컨셉이 나왔는지 — 없으면 이 줄 생략}
⚠️ 미결: {발견한 충돌·미결 사항 — 없으면 이 줄 생략}

참조 이미지: {경로1} ({무엇을 위한 참고인지})
          + {경로2} (...)

프롬프트:
(색감·테마·무드는 아래 문단들 합쳐서 반드시 포함 — 별도 「조명 연출」 문단은 선택)

{에셋 고정 문단}

{장면/구도 문단 — 색감·무드를 여기 넣어도 됨}

{조명 연출 문단 — 선택. 시간대·광원·그림자만. 생략해도 색감·테마·무드는 위/아래에 남아 있어야 함}

{스타일 앵커 문장 — 프로젝트 테마·레퍼런스 톤·무드}

{기술 조건 한 줄}
```

여러 버전(시간대·감정 등)이 필요하면:

```
공통 베이스 프롬프트(모든 버전 동일, 아래 버전별 문장만 교체):
{공통 부분}

--- {버전1 이름} ---
{버전1 전용 문장}

--- {버전2 이름} ---
{버전2 전용 문장}
```

## NN-{컨셉명}_comfyui.txt 스켈레톤 (ComfyUI, 요청 시에만)

```
[컨셉] {컨셉명} (ComfyUI)
참조: Nano Banana 버전과 동일 컨셉 — 01-{컨셉명}.txt

Positive:
{에셋 고정 → 장면 → 색감·테마·무드(필수) → 스타일 → (조명 연출, 있을 때만)}

Negative:
{배제할 것 — 스타일 확정 사항에서 역산. 기본 공통: text, watermark, blurry, extra limbs, deformed}

참고: ComfyUI는 이미지 레퍼런스를 프롬프트에 직접 못 넣는다. 캐릭터 정체성을
유지하려면 IPAdapter 등 별도 노드로 참조 이미지를 물려야 한다 (이 스킬 범위 밖).
```

## 실제 예시 (이 프로젝트에서 이미 쓴 것)

에셋 고정 → 장면/구도 → 스타일 앵커(테마·무드) → 기술 조건. **색감·테마·무드는 항상** 본문에 포함. 시간대·광원은 무드 시리즈일 때만 별도 「조명 연출」 문단 추가.
