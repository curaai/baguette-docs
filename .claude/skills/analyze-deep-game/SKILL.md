# 외부 게임 심층 분석

외부 게임 자체를 먼저 분석.
우리 게임 내용을 본문에 섞지 않음.

## 시작 전

필수 입력:

- 게임 이름
- 분석할 컨텐츠·시스템 1개

대상이 넓으면 먼저 질문.

## 조사 내용

1. 플레이어가 실제로 보는 장면.
2. 플레이어 행동.
3. 게임 반응과 보상.
4. 반복 흐름 안에서 맡는 역할.
5. 비슷한 게임과 다른 점.
6. 만드는 감정·피로·학습 부담.
7. 확인한 자료와 추정 구분.

## 심층 분석 문서

저장 경로:

`planning/references/<게임>-<주제>-deep.md`

구조:

```markdown
# [게임] [주제] 심층 분석

## 실제 모습
[플레이어가 보는 장면과 행동]

## 작동 방식
[조건, 반응, 보상, 반복 흐름]

## 다른 게임과 차이
[비슷한 사례와 핵심 차이]

## 플레이 경험
[감정, 반복 피로, 배우기 어려운 부분]

## 확인한 자료
[직접 플레이 여부, 버전, 출처]
```

## 우리 게임 비교 확인

심층 분석 저장 후 `AskUserQuestion` 실행.

질문:

`우리 게임과 비교할 부분을 선택해 주세요.`

분석에서 발견한 주요 항목 3~5개를 선택지로 제시.
마지막 선택지: `비교하지 않음`.
여러 항목 선택 허용.

`비교하지 않음` 선택 시 종료.
항목 선택 시 별도 비교 문서 생성.

## 우리 게임 비교 문서

저장 경로:

`planning/references/<게임>-<주제>-compare.md`

선택한 항목만 비교.
심층 분석 본문 복사 금지.

구조:

```markdown
# [게임] [주제] 우리 게임 비교

## 비교할 부분
[사용자가 선택한 항목]

## 외부 게임 모습
[외부 게임에서 실제로 보이는 방식]

## 우리 게임 선택
[우리 게임에 맞는 선택지 2~3개]

## 주의할 점
[톤, 범위, 반복 피로, 제작 부담]

## 확인한 자료
[심층 분석 파일과 추가 출처]
```

## 작성 규칙

- 짧고 정확한 문장 사용.
- 확인한 사실과 추정 구분.
- 내부 코드명 금지.
- 출처 없는 수치 단정 금지.
- 심층 분석과 우리 게임 비교를 같은 파일에 작성 금지.
- 비교 문서는 사용자 선택 없이 생성 금지.
