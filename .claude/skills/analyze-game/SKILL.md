# 레퍼런스 브리핑

목적: 좋은 분석 보고서 작성 아님.
목적: 사용자가 우리 게임 방향 선택 가능하게 만들기.

기본 분량: 한 페이지 이내.
깊은 게임 분석 요청은 `analyze-deep-game` 사용.

## 시작 전

다음 내용 확인.

- 이번에 결정할 것 1개
- 조사할 게임 1~3개
- 궁금한 장면·기능

결정 대상 불명확하면 먼저 질문.

## 조사 순서

1. 게임에서 실제로 보이는 장면 확인.
2. 플레이어 행동 확인.
3. 게임 반응 확인.
4. 사례 사이 핵심 차이 확인.
5. 우리 게임 선택지 2~3개 작성.
6. 사용한 자료 기록.

## 문서 구조

```markdown
# [주제] 레퍼런스

## 왜 보는가
[이번에 결정할 내용. 1~2문장.]

## 실제 모습

**[게임 이름]**
- 플레이어가 보는 것
- 플레이어가 하는 것
- 게임 반응
- 이 방식의 장점·불편

## 우리 선택

1. **[쉬운 선택지 이름]**
   - 어떤 모습인지
   - 좋은 점
   - 주의할 점

## 확인한 자료

- [게임·버전] — 직접 플레이 또는 관찰 — [자료 링크]
```

## 작성 규칙

- 짧고 쉬운 한국어 사용.
- 게임 속 장면부터 설명.
- 선택지는 내부 코드명 없이 작성.
- 확인한 사실과 추정 구분.
- 직접 플레이 여부 표시.
- 출처 링크는 마지막 장에만 정리.
- 표는 문장보다 쉬울 때만 사용.

## 쓰지 않을 것

- Chapter 번호
- 코어 루프·컨텐츠 루프 정의
- 동기·긴장·보상·정보·분기·경합 목록
- 다이어그램
- 피로도·학습 곡선 의무 분석
- 내부 단계명
- A, B, I1 같은 선택지 코드
- 같은 결론 반복
- 기획 선택과 무관한 게임 전체 설명
