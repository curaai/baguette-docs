# 게임 흐름 기획

게임에서 행동이 어떻게 이어지고 반복되는지 설계한다.
컨텐츠 하나의 세부 규칙은 다루지 않는다.

## 설계 순서

1. 플레이어가 가장 자주 하는 행동을 찾는다.
2. 그 행동의 결과와 다음 행동이 이어지는 이유를 정한다.
3. 한 번 게임을 켰을 때의 시작, 진행, 마무리를 정한다.
4. 게임을 끈 뒤 다시 돌아오고 싶은 이유를 정한다.
5. 흐름이 끊기거나 같은 행동만 강요되는 곳을 고친다.

## 확인할 내용

- 처음 켰을 때 무엇을 바로 할 수 있는가?
- 짧게 반복해도 기분 좋은 행동은 무엇인가?
- 행동의 결과가 다음 행동으로 자연스럽게 이어지는가?
- 한 번의 플레이가 억지로 끊기지 않고 마무리되는가?
- 다음에 다시 켰을 때 보고 싶은 변화가 있는가?
- 아무것도 할 수 없는 시간이 생기지 않는가?

핵심 선택이 비어 있으면 사용자에게 묻는다.
정해지지 않은 내용을 임의로 채우지 않는다.

## 문서 작성

설계가 끝나면 `game-planning` 스킬을 사용해 기획서로 작성한다.
문서 형식과 문장 규칙은 이 스킬에 중복해서 적지 않는다.
