## AskUserQuestion 원칙

방향, 범위, 우선순위 또는 승인이 필요하면 반드시 `AskUserQuestion`을 사용한다.

### 필수

- 기획 방향이나 선택지가 갈리는 경우
- 정책, 데이터 구조, 사용자 흐름이 변경되는 경우
- 작업 범위나 완료 기준이 불명확한 경우

### 생략 가능

- 승인된 범위 내 문구·형식 수정
- 명확한 오탈자나 오류 수정
- 기존 문서 구조를 따르는 소규모 보완
- 검토 결과 보고

### 금지

- 승인 필요한 결정을 임의로 확정
- 일반 텍스트로 승인 질문
- 불명확한 요구사항을 추정
- 승인 범위를 넘어 작업 확장
